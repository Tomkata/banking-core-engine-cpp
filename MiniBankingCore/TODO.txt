Visitor pattern работи така:
Имаш йерархия от класове (CheckingAccount, SavingsAccount, DepositAccount) и искаш да правиш различни операции върху тях без да добавяш методи в самите класове.
Идеята:
Account казва: "ето ме, прави каквото искаш с мен"
Visitor казва: "ако си DepositAccount → правя X, ако си SavingsAccount → правя Y"
Конкретно:
cpp// Account само "отваря вратата"
class Account {
    virtual void Accept(IAccountVisitor& visitor) = 0;
};

// DepositAccount казва "ето ме съм DepositAccount"
void DepositAccount::Accept(IAccountVisitor& visitor) {
    visitor.Visit(*this);  // подава себе си
}

// SqliteAccountRepository е visitor-ът
class SqliteAccountRepository : public IAccountVisitor {
    void Visit(const DepositAccount& acc) {
        // INSERT INTO deposit_accounts
    }
    void Visit(const SavingsAccount& acc) {
        // INSERT INTO saving_accounts
    }
};
Вместо if (type == Deposit) → полиморфизмът сам избира правилния Visit.
